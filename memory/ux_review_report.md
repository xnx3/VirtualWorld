## 优点分析

### 1. 一键安装流程设计良好

**文件**: `MainActivity.kt:429-518` (`runGenesisOneTap`)

- 真正的"一键"体验：自动完成 Termux 检测 -> 存储权限检查 -> 文件准备 -> Termux 安装 -> 启动服务
- 清晰的状态反馈：`success`, `running`, `bundledRuntime` 等字段让用户知道当前状态
- 智能等待机制：有 bundle 时等待 75 秒，无 bundle 时等待 20 秒（合理的时间预期）

### 2. 状态指示清晰

**文件**: `settings_screen.dart:1011-1095`

- 三步状态指示器（Step 1: Termux / Step 2: 安装文件 / Step 3: 服务）
- 醒目的颜色区分：绿色=完成，橙色=待处理，灰色=未开始
- 主按钮状态动态变化："一键安装并启动 Genesis" vs "Genesis 已运行，刷新状态"

### 3. API 配置便捷

**文件**: `settings_screen.dart:48-69`

- 预设提供商快速选择（OpenAI、DeepSeek、Claude、Ollama）
- API Key 可隐藏显示，保护隐私
- 保存前自动测试连接，避免配置错误

### 4. 文件指纹升级机制

**文件**: `GenesisInstaller.kt:517-535`

- 使用 SHA256 指纹检测资产变更
- 自动触发升级安装，用户无需手动判断版本

### 5. 安装脚本回退机制

**文件**: `quick_install.sh:158-161`

- Bundle 文件缺失时自动回退到完整安装
- 用户无需理解两种安装方式的区别

## 问题与改进建议

### 问题1: 安装阶段概念混淆 [严重程度: 中]

**位置**: `GenesisInstaller.kt:237-277` (`buildInstallMessage`)

**问题描述**:
- 用户点击"一键安装"后，看到的是"安装文件已准备就绪"
- 但"准备文件"和"真正安装"是两个不同阶段
- 普通用户可能误解为安装已完成

**用户视角**:
```
用户操作: 点击"一键安装并启动 Genesis"
系统反馈: "安装文件已准备就绪"
用户理解: "安装完成了？" ← 混淆点
实际状态: 文件只是复制到共享目录，Termux 里还没开始安装
```

**建议改进**:
- 将 `installGenesis` 的成功消息改为"安装文件已准备好，正在 Termux 中完成安装..."
- 或者直接使用 `runGenesisOneTap` 作为唯一入口，避免用户看到中间状态

---

### 问题2: 错误提示不够具体 [严重程度: 中]

**位置**: `MainActivity.kt:403-411` (SecurityException 处理)

**问题描述**:
```
错误信息: "需要 Termux RUN_COMMAND 权限。请先打开 Termux 并允许外部应用调用后重试"
```

- 没有告诉用户具体如何在 Termux 中授权
- 用户可能不知道 RUN_COMMAND 权限是什么

**建议改进**:
```
错误信息: "Termux 需要授权外部应用调用权限。
解决步骤：
1. 打开 Termux 应用
2. 在 Termux 设置中找到 '外部应用调用' 或 'RUN_COMMAND'
3. 开启该权限
4. 返回 Genesis 应用重试"
```

---

### 问题3: 排障按钮文案晦涩 [严重程度: 低]

**位置**: `settings_screen.dart:1035`

**问题描述**:
```
按钮文案: "仅重新准备安装文件（排障）"
```

- "排障"术语对普通用户不友好

**建议改进**:

```
提示文案: "正常情况请点击上面的主按钮。此按钮仅在安装失败时使用。"
```

### 问题4: Bundle 回退提示不明显 [严重程度: 低]

**位置**: `quick_install.sh:158-161`

**问题描述**:
```bash
if [ -z "$BUNDLE_URL" ] && [ -z "$BUNDLE_FILE" ] && [ -f "$SCRIPT_DIR/install.sh" ]; then
    echo -e "${YELLOW}Bundle file not found. Falling back to full installer...${NC}"
    bash "$SCRIPT_DIR/install.sh"
    exit $?
fi
```

- 回退提示用黄色输出，可能被用户忽略
- 用户不知道为什么安装突然变慢（完整安装需要 20 分钟）

**建议改进**:
```bash
echo -e "${RED}========================================${NC}"
echo -e "${YELLOW}注意: Bundle 文件未找到，正在使用完整安装方式${NC}"
echo -e "${YELLOW}完整安装需要约 20 分钟，请耐心等待...${NC}"
echo -e "${RED}========================================${NC}"
```

### 问题6: 存储权限提示时机 [严重程度: 低]

**位置**: `settings_screen.dart:406-444`

**问题描述**:
- 存储权限请求后显示对话框，需要用户手动点击"已授权，继续"
- 更好的做法是自动轮询权限状态，授权后自动继续

**建议改进**:
```dart
// 请求权限后，每秒检查权限状态
await _channel.invokeMethod('requestStoragePermission');

// 自动轮询
for (int i = 0; i < 30; i++) {
  await Future.delayed(Duration(seconds: 1));
  final hasPermission = await _channel.invokeMethod('hasStoragePermission');
  if (hasPermission) {
    _doInstallGenesis();  // 自动继续
    return;
  }
}
// 超时后才显示手动对话框
```

### 问题7: 安装进度百分比不够精确 [严重程度: 低]

**位置**: `GenesisInstaller.kt:139-204`

**问题描述**:
- 进度百分比是人工估算的（5%, 10%, 20%, 60%, 80%, 100%）
- 复制 genesis 源码（步骤4）从 20% 直接跳到 60%，跨度太大
- 用户可能觉得进度卡住了

**建议改进**:
- 基于实际文件数量或大小计算进度
- 或者在跨度大的步骤中添加子进度反馈
